# Lecture 1 – Introduction to Java

---

## 1. What is Java?

Java is a **High-Level, Object-Oriented, Platform-Independent Programming Language** developed by **James Gosling** at **Sun Microsystems** in **1995**.

Java was designed with the principle of **"Write Once, Run Anywhere (WORA)"**, which means a Java program can run on different operating systems without modifying the source code.

Today, Java is one of the world's most popular programming languages and is widely used for developing:

- Enterprise Applications
- Android Applications
- Web Applications
- Desktop Applications
- Cloud-Based Systems

---

## 2. Why is Java Popular?

Java is widely used because it provides several advantages:

-  Easy to Learn and Read
-  Platform Independent
-  Object-Oriented
-  Secure
-  Robust
-  Portable
-  High Performance
-  Multithreaded
-  Huge Community Support
-  Rich Standard Library

These features make Java suitable for developing both small and large-scale applications.

---

# 3. Features of Java

## 3.1 High-Level Language

Java uses simple and readable syntax that is easy for humans to understand.

Unlike machine language, programmers can easily write, debug, and maintain Java programs.

---

## 3.2 Object-Oriented

Java follows the **Object-Oriented Programming (OOP)** paradigm.

It organizes programs using **Classes** and **Objects**, making the code modular, reusable, and easier to maintain.

---

## 3.3 Platform Independent

Java programs can run on multiple operating systems without changing the source code.

This is possible because Java programs are executed by the **Java Virtual Machine (JVM)**.

This feature is known as **Write Once, Run Anywhere (WORA)**.

---

## 3.4 Secure

Java provides several built-in security mechanisms such as:

- Bytecode Verification
- Class Loader
- Security Manager (Legacy Concept)
- No Direct Pointer Manipulation

These features make Java suitable for banking, financial, and enterprise applications.

---

## 3.5 Robust

Java is considered **Robust** because it provides:

- Strong Memory Management
- Exception Handling
- Garbage Collection
- Compile-time and Runtime Error Checking

These features help in developing reliable applications.

---

## 3.6 Portable

Java programs can be easily moved from one operating system to another without requiring modifications.

---

## 3.7 High Performance

Although Java is an interpreted language, it provides good performance using the **Just-In-Time (JIT) Compiler**, which converts frequently used bytecode into native machine code.

---

## 3.8 Multithreaded

Java supports multithreading, allowing multiple tasks to execute simultaneously.

This improves application performance and responsiveness.

---

## 3.9 Rich Standard Library

Java provides thousands of built-in classes and packages for:

- Input and Output (I/O)
- Networking
- Collections
- File Handling
- Database Connectivity
- GUI Development

These libraries reduce development time.

---

## 3.10 Huge Community Support

Java has one of the largest developer communities in the world.

Developers can easily find:

- Tutorials
- Documentation
- Libraries
- Frameworks
- Community Support

---

# 4. JDK (Java Development Kit)

The **Java Development Kit (JDK)** is the complete software package required to develop Java applications.

### It contains:

- Java Compiler (`javac`)
- Java Launcher (`java`)
- Java Runtime Environment (**JRE**)
- Java Virtual Machine (**JVM**)
- Development Tools

### Purpose

- Write Java Programs
- Compile Java Programs
- Run Java Programs

> **Without the JDK, Java applications cannot be developed.**

---

# 5. JRE (Java Runtime Environment)

The **Java Runtime Environment (JRE)** provides the environment required to execute Java applications.

### It contains:

- Java Virtual Machine (**JVM**)
- Java Class Libraries
- Runtime Files

### Purpose

- Run Java Applications

> **Without the JRE, Java programs cannot execute.**

---

# 6. JVM (Java Virtual Machine)

The **Java Virtual Machine (JVM)** is responsible for executing Java bytecode.

### Functions of JVM

- Converts Bytecode into Machine Code
- Executes Java Programs
- Provides Platform Independence
- Performs Garbage Collection
- Manages Memory
- Ensures Security

> Every operating system has its own JVM implementation.
